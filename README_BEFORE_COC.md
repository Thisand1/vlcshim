# Before reading the Code of Conduct

This repository used to have a deliberately abrasive README. That is no longer the direction of the project.

## What to expect

1. Documentation should be direct and readable.
2. Issues and pull requests should stay respectful and specific.
3. Feedback about bugs, setup problems, and documentation gaps is welcome.
4. If something is unclear, open an issue and describe the exact problem, your Windows build, and your VLC setup.

## TL;DR

Read the docs carefully, be specific when reporting problems, and keep discussion constructive.
